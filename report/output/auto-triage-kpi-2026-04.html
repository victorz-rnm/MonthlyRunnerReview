2. KPI Definitions

Column definitions
Term
Name of the KPI term, abbreviation, or report convention.
Meaning
Plain-language definition used by the report and presenter.
TermMeaning
ppPercentage points. Example: if a KPI moves from 12.0% to 10.0%, the delta is -2.0pp. It is a direct subtraction of two percentages, not a relative percent change.
First auto p50Median time from incident creation to the first auto-triage action. The source KQL stores this in seconds as FirstAutoActionLatencySeconds. The HTML converts seconds to a readable unit, so 2.2m means 2.2 minutes.
LatencyFirstAutoActionAt is the earlier of first auto transfer and first auto link. Latency is datetime_diff('second', FirstAutoActionAt, OriginalCreateDate). Transfer and link latency use the same calculation with FirstAutoTransferAt or FirstAutoLinkAt.
Team / Overall windowTeamLevelKpi and OverallKpi cover the rendered report window: 2026-03-01 to 2026-04-30. For this report that is previous month plus current month, not the source query's original 90-day lookback.
Report targetsTargets are report-side defaults, not values from the KQL result sets: coverage 15%, correctness 90%, max rework 10%, and first-auto-action p50 latency 30 minutes. They can be changed with CLI flags such as --coverage-target.
ReworkA rework row is counted only when the matching auto action was later corrected: AutoTransferCorrect = 0 or AutoLinkCorrect = 0. This keeps rework consistent with the transfer/link correctness metrics.

  • Abbreviations: KPI = Key Performance Indicator; pp = percentage points; p50 = 50th percentile / median; p95 = 95th percentile; s, m, h, and d = seconds, minutes, hours, and days; auto = auto-triage action.
  • The renderer bounds lookBackRange to the previous-period start and adds reportEndTime to incident create/history filters, so all three result sets cover the same report window.
  • Displayed rework rates use matching-action correctness flags, not the broader manual-after-first-auto support fields.
  • IncidentLevelKpi is still queried and cached because Overview, Monthly Summary, latency, and rework details are calculated from it. The full incident-level table is intentionally not rendered in the HTML.
  • Correctness follows the source query: later manual transfer after auto transfer means transfer incorrect; later manual relationship update after auto link means link incorrect.
